## 三、功能需求（原始 PDF 規格）

### 3.1 作業位置

於 `http://localhost:5174/portfolios` 頁面，於各個證券明細下方的區塊新增「資產總計」功能。

### 3.2 作業畫面規格

| 項目 | 說明 |
|------|------|
| 甲、作業畫面 | 表格位於投資組合清單下方，含占比標頭與資產明細表 |
| 乙、序號 | 自動增加，不可手動輸入 |
| 丙、帳戶類型 | 選單（combobox），可手動輸入；輸入新值後自動加入選單清單 |
| 丁、市值 | 手輸數字 |
| 戊、mark | 手輸備註文字 |
| 己、帶入證券明細 | 可勾選上方已建立的【XX-證券明細】自動帶入資料；帶入時帳戶類型預設為「證券帳戶」 |

### 3.3 表格欄位定義

| 欄位 | 型態 | 說明 |
|------|------|------|
| 序 | 自動 | 依列順序自動編號（1, 2, 3…），唯讀 |
| 帳戶類型 | combobox | 預設值：證券帳戶、銀行帳戶；可新增自訂類型 |
| 市值 | 數字手輸 | 手動輸入或從證券明細自動帶入 |
| mark | 文字手輸 | 自由備註；從證券明細帶入時自動填入「XX-證券明細」 |
| 股票-個股 | 唯讀 | 從連結之投資組合 by_asset_type 自動計算（mv_2） |
| 股票ETF | 唯讀 | 同上 |
| 債券-ETF | 唯讀 | 同上 |
| CB | 唯讀 | 同上 |
| CBAS | 唯讀 | 同上 |
| 小計 | 自動 | 上述五個資產類別之合計 |

### 3.4 彙整標頭

| 項目 | 計算邏輯 |
|------|---------|
| 證券占比 | 帳戶類型 = 「證券帳戶」的市值總和 ÷ 全部市值總和 |
| 現金占比 | 帳戶類型 ≠ 「證券帳戶」的市值總和 ÷ 全部市值總和 |
| 資產總計 | 全部列的市值加總 |

## 四、實作記錄

### 4.1 修改檔案

僅修改前端一個檔案：
```
frontend/src/views/HomeView.vue
```

後端無需修改。

### 4.2 資料儲存方式

使用 `localStorage` 持久化，不需後端 API 變更：

| localStorage Key | 內容 |
|-----------------|------|
| `ta_rows_v1` | 資產總計列資料（JSON array） |
| `ta_types_v1` | 帳戶類型清單（JSON array） |

### 4.3 資料結構

```typescript
interface TotalRow {
  id: number                                  // 唯一識別碼
  accountType: string                         // 帳戶類型
  marketValue: number | null                  // 市值
  mark: string                                // 備註
  linkedPortfolioId: number | null            // 連結的投資組合 ID（帶入時設定）
  breakdown: Partial<Record<AssetCol, number>> // 資產類別明細（來自投資組合）
}
```

### 4.4 主要功能說明

#### (1) 行內新增列（Inline New Row）

- 表格最底部常駐一行綠色虛線輸入列（與證券明細概覽相同設計）
- 輸入帳戶類型、市值、mark 後，按 **Enter** 或點擊 **＋** 按鈕儲存
- 儲存後輸入列自動清空，等待下一筆輸入

#### (2) 帳戶類型 Combobox

- 使用 `<input list>` + `<datalist>` 實作原生 combobox
- 預設選項：`證券帳戶`、`銀行帳戶`
- 輸入欄位 blur 時，若輸入值不在清單中，自動加入 `acctTypes` 並持久化

#### (3) 從證券明細帶入（多筆）

- 點擊「＋ 從證券明細帶入」開啟 picker 彈窗
- 彈窗顯示所有投資組合清單：
  - 未帶入者：顯示 checkbox，可勾選
  - 已帶入者：顯示綠色 ✓「已帶入」，不可重複選取
- 勾選多筆後，點擊「帶入已選取（N）」一次批次帶入
- 帶入時呼叫 `api.getPortfolio(pid)` 取得明細，自動填入：
  - `accountType` = `'證券帳戶'`
  - `marketValue` = `totals.mv_2`
  - `mark` = `'{portfolio.name}-證券明細'`
  - `breakdown` = `by_asset_type[*].mv_2`

#### (4) 刪除列

- 每列右側「刪」按鈕，確認後移除
- 刪除投資組合時，同步移除關聯的資產總計列

#### (5) 合計列（tfoot）

- 市值合計、各資產類別合計、小計合計
- 即時計算，不需存檔

### 4.5 關鍵 computed 屬性

```typescript
// 證券帳戶市值合計
const secTotal = 帳戶類型 === '證券帳戶' 的 marketValue 加總

// 現金（非證券）帳戶市值合計
const cashTotal = 帳戶類型 !== '證券帳戶' 的 marketValue 加總

// 占比（四捨五入至小數點後兩位）
const secPct  = secTotal / grandTotal * 100
const cashPct = cashTotal / grandTotal * 100

// 資產類別欄合計
const colTotals = 各 ASSET_COL 的 breakdown 加總

// 小計欄合計
const grandSubtotal = colTotals 各欄加總
```
